# Team Writing Rules

Shared rules for any text produced in this project: UI copy, docs, comments,
commit messages, and Kiro's own replies. These are committed to the repo, so
every coworker's Kiro sessions follow the same rules.

## How to add a rule

Copy the block below, give it the next number, and fill it in. Keep each rule
short and specific. If a rule ever conflicts with another, the lower number
wins.

```
### <number>. <short title>
- **Rule:** <what to do or not do>
- **Instead:** <the preferred alternative, if the rule is a "don't">
- **Example:** <optional short before/after>
```

## Rules

### 1. Never use the em dash character
- **Rule:** Do not use the "—" character anywhere in text.
- **Instead:** Use a comma, a period, or rewrite the sentence into two sentences. A plain hyphen "-" is fine when joining words.
- **Example:** Before: "It works fast — really fast." After: "It works fast, really fast."

### 2. Always use natural human language
- **Rule:** Write the way a person actually talks. Avoid stiff corporate or robotic phrasing, filler, and hype words.
- **Instead:** Say things plainly and directly. Cut words that do not add meaning.
- **Example:** Before: "We leverage cutting-edge synergies." After: "We use tools that work well together."
